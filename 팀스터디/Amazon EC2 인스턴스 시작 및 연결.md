# Amazon EC2 인스턴스 시작 및 연결

* 시작하기 전에 알아야할 것
  * EC2(Elastic Compute Cloud): 클라우드에서 가상 머신을 만들고 실행하는데 사용하는 아마존 웹서비스이다. 즉, 물리적인 서버를 구축할 필요없이 클라우드 상의 서버를 할당받아 사용하는 것을 말한다.
  * 인스턴스: 가상의 컴퓨팅 환경이다. (컴퓨터 1대라고 생각하면 편하다)
  * EBS(Elastic Block Store): Amazon EC2 인스턴스에 사용할 지속적인 블록 등급 스토리지 볼륨을 제공, 필요한 용량에 맞게 구입 가능
  * AMI (Amazon Machine Image): 서버에 필요한 운영체제와 어려 소프트웨어 들이 적절히 구성된 템플릿으로  인스턴스를 시작하는 데 필요한 정보를 제공한다. 인스턴스를 시작할 때 AMI를 지정해야 한다. 동일한 구성의 인스턴스가 여러 개 필요할 때는 한 AMI에서 여러 인스턴스를 시작할 수 있다. 또한, 서로 다른 구성의 인스턴스가 필요할 때는 다양한 AMI를 사용하여 인스턴스를 시작해야 한다. (ex. Window, Ubuntu, Linux 등)

=> 이 인스턴스는 Amazon EBS 지원 인스턴스(루트 볼륨이 EBS 볼륨임을 의미)이다. 인스턴스가 실행되는 가용 영역을 지정하거나 적합한 가용 영역이 Amazon EC2에서 자동으로 선택할 수 있다. 인스턴스를 시작할 때 키 페어와 보안 그룹(방화벽을 생각하면 편하다)을 지정하여 인스턴스 보안을 설정한다. 인스턴스에 연결할 때는 인스턴스 시작 시 지정한 키 페어의 프라이빗 키를 지정해야 한다.



## 1단계: 인스턴스 시작

1. 콘솔 대시보드에서 **인스턴스 시작**을 선택한다.

![image-20200115112822883](images/image-20200115112822883.png)

=> aws EC2에 접속하면 `인스턴스 시작` 버튼을 눌러 간편하게 시작할 수 있다.



2. **Amazon Machine Image(AMI) 선택** 페이지에 인스턴스에 대한 템플릿 역할을 하는 *Amazon Machine Image(AMI)*라는 기본 구성 목록이 표시된다. Amazon Linux 2의 HVM 버전을 선택한다. 해당되는 AMI는 "프리 티어 사용 가능"으로 표시된다. 

![image-20200115130118616](images/image-20200115130118616.png)

=> 인스턴스 시작 버튼을 누르면 가장 상단에 나타나는 Amazon Linux 2의 HVM 버전 AMI를 선택한다.



3. **인스턴스 유형 선택** 페이지에서 인스턴스의 하드웨어 구성을 선택할 수 있습니다. 기본적으로 선택된 `t2.micro` 유형을 선택합니다. 이 인스턴스 유형은 프리 티어에 적격이다.

![image-20200115130340175](images/image-20200115130340175.png)

*AWS 프리티어(Free Tier)는 1년간 무료로 사용할 수 있는 새로운 AWS 계정용 특정 리소스의 양 및 유형을 만들어 준다.



4. **검토 후 시작(Review and Launch)**을 선택하여 마법사가 다른 구성 설정을 완료하게 한다.

![image-20200115131046631](images/image-20200115131046631.png)



5. **인스턴스 시작 검토** 페이지의 **보안 그룹** 아래에서 마법사가 보안 그룹을 만들고 선택했음을 확인한다. 이 보안 그룹을 사용하거나, 다음 단계를 이용하여 설정을 시작할 때 만든 보안 그룹을 선택한다.

   

6. **보안 그룹 편집**을 선택한다. **보안 그룹 구성** 페이지에서 **Select an existing security group(기존 보안 그룹 선택)**이 선택되어 있는지 확인한다.

![image-20200115151257035](images/image-20200115151257035.png)

*인바운드 규칙

![image-20200115151437440](images/image-20200115151437440.png)

=> 보안 그룹은 인스턴스에 대한 트래픽을 제어하는 방화벽 규칙 세트이다. 이 페이지에서는 특정 트래픽을 인스턴스에 도달하도록 허용할 규칙을 추가할 수 있다. 예를 들면 웹 서버를 설정하여 인터넷 트래픽을 인스턴스에 도달하도록 허용하려는 경우 HTTP 및 HTTPS 트래픽에 대한 무제한 액세스를 허용하는 규칙을 추가한다.



7. 기존 보안 그룹 목록에서 보안 그룹을 선택한 다음 **검토 후 시작(Review and Launch)**을 선택한다.

   

8. **인스턴스 시작 검토** 페이지에서 **시작**을 선택한다.

   

9. **새 키 페어 생성**을 선택하고 키 페어 이름을 입력한 다음 **키 페어 다운로드**를 선택한다. 사용자가 프라이빗 키 파일을 저장할 수 있는 유일한 기회이므로 반드시 다운로드한다. 프라이빗 키 파일은 안전한 장소에 저장한다. 인스턴스를 시작할 때 키 페어의 이름을 제공하고, 인스턴스에 연결할 때마다 해당 프라이빗 키를 제공해야 한다. 준비되면 승인 확인란을 선택한 다음 **인스턴스 시작**을 선택한다.

![image-20200115133843825](images/image-20200115133843825.png)



9. 확인 페이지에서 인스턴스가 실행 중인지 확인할 수 있다. **인스턴스 보기**를 선택하여 확인 페이지를 닫고 콘솔로 돌아간다.

![image-20200115150834573](images/image-20200115150834573.png)



10. **인스턴스** 화면에서 시작 상태를 볼 수 있다. 인스턴스를 시작하는 데 약간 시간이 걸린다. 인스턴스를 시작할 때 초기 상태는 `pending`이다. 인스턴스가 시작된 후에는 상태가 `running`으로 바뀌고 퍼블릭 DNS 이름을 확인한다. (**퍼블릭 DNS(IPv4)** 열이 숨겨져 있는 경우 페이지 오른쪽 상단 모서리에 있는 **열 표시/숨기기**(기어 모양 아이콘)를 선택한 다음 **퍼블릭 DNS(IPv4)**를 선택한다.)

    

11. 연결할 수 있도록 인스턴스가 준비될 때까지 몇 분 정도 걸릴 수 있다. 인스턴스가 상태 확인을 통과했는지 확인한다.
